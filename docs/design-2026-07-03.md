# Design — Code Quest Series (approved 2026-07-03)

The approved design that turns Code Quest from a single course into a recurring series.

## Goal

Extend Code Quest (a completed single-file course teaching JS fundamentals through a tiny
Zelda-style game) into a **series of self-contained HTML levels** that progressively teach
advanced game programming and build toward a shared multi-floor, multi-level adventure with
enemies, rewards, objectives, and a boss.

## Decisions

1. **Series scope:** 4 new levels (L2–L5). Each is one self-contained HTML file covering a
   themed batch of concepts, ending in a bigger milestone of the shared game.
2. **Game continuity:** From L2 forward it is **one evolving game**, re-envisioned with
   richer visuals. Each level = the previous game + one major new system. L1 game was a
   single square hero; L2+ uses pixel-sprite characters (more engaging, still offline).
3. **Standards library, not a runtime dependency:** A `STANDARDS.md` rulebook + a master
   `template/base.html` (theme, world-map sidebar, sandboxed playground with canvas,
   localStorage save engine, sounds, certificate, hard-word tooltips) + `template/sprites.md`.
   Levels are built by copying the template and inlining — every shipped file stays 100%
   self-contained and offline.
4. **Level 1 frozen:** Moved into `levels/level-01/index.html` unchanged.
5. **Human-readable variable names (hard rule):** All learner-facing code uses meaningful,
   spelled-out names (`nextCol` not `nc`, `heroHealth` not `hp`). `ctx` is the one
   deliberate exception (the real Canvas API name). Codified in `STANDARDS.md §2`.

## Visual approach

Characters are **pixel sprites drawn from a grid array of color codes**, scaled up on the
canvas — no image files. This is more engaging than L1's single square *and* reinforces
arrays + nested loops. The hero faces 4 directions with a walk wiggle and sword swing;
enemies and the boss each have distinct sprites. Format + palette + helper in
`template/sprites.md`; the `drawSprite` helper and shared `PALETTE` are exposed inside the
playground sandbox by `template/base.html`.

## Teaching contract (carried into every level)

5 parts per topic, in order: Lesson → Mental Model → Common Mistakes → Exercise → Recap.
Simple English, every hard word explained, real-world analogies, side-by-side comparisons,
playground-preloaded exercises, heart-meter progress, rupees, sounds, and a completion
certificate. Full rules in `STANDARDS.md`.

## Deliverable for this pass

Repo + private GitHub setup, the standards library, L1 moved in, and this plan — committed
and pushed. Building L2–L5 course files is the next step, starting with L2.
